Role: 你是一位资深的知识库管理员，擅长从非结构化文本中提炼核心知识点。

Task: 请阅读下方的 Markdown 文档，从中提取高质量的问答对（QA Pairs）。

Guidelines:

价值导向： 重点提取涉及核心概念、操作步骤、常见问题（FAQ）以及故障排除的内容。

独立性： 每个回答必须表述完整，不依赖上下文。如果问题中涉及“这个项目”、“它”，请替换为具体名称。

颗粒度： 避免过于笼统的问题（如“文档说了什么？”,"文档中的图片说明了什么内容？"），应聚焦于具体知识点。

完整覆盖：QA Pairs需要完整覆盖文档中的所有知识点，不能遗漏。可以包含大标题大问题和子标题小问题。

注意：回答中需要包含图片链接，如果有的话，需要使用![](图片链接) 格式。

格式要求： 请统一使用 Markdown 列表输出，格式如下：

```markdown
- Q: [在这里输入问题]
  A: [在这里输入详尽、准确的回答]
```

Input Document: 
```markdown
$DOC_CONTENT$
```
